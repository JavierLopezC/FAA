def error_sk(data, pred):
    tot = 0
    for i in range(0, len(pred)):
        if pred[i] != data[i]:
            tot += 1
    tot /= len(pred)
    #print("error de: " + str(tot))
    return tot

def valores_roc(data, pred):
    fp = 0
    fn = 0
    tp = 0
    tn = 0
    for i in range(0, len(pred)):
        if pred[i] == data[i]:
            if pred[i] == 1:
                tp += 1
            elif pred[i] == 0:
                tn += 1
        elif pred[i] != data[i]:
            if pred[i] == 1:
                fp += 1
            elif pred[i] == 0:
                fn += 1
    return tp, tn, fp, fn

Probamos nuestro clasificador Naive-Bayes con validacin simple. Este tipo de particionado divide la lista en dos partes aleatorias de un tamao determinado, una para entrenar y otra para probar las predicciones que hemos hecho. realizamos este proceso y las respectivas pruebas 100 veces y nos quedamos con los valores medios de error, ya que de hacerlo una nica vez el resultado estara demasiado sesgado por el resultado de la separacin aleatoria. La principal ventaja respecto a la validacin cruzada es que de esta forma la divisin de datos entrenamiento-prueba no depende del nmero de iteraciones. Por otro lado con este mtodo hay algunas muestras que pueden quedar sin evaluar y otras que pueden ser evaluadas ms de una vez.

Obtenemos dos resultados: con y sin correccin de Laplace. La correccin de laplace nos evita caer en el error de hacerle "creer" a Naive-Bayes que porque no tengamos ningn ejemplo de pertenencia a una clase dados una serie de valores, no significa que la probabilidad sea automticamente 0.         fprl = fpl / (fpl + tnl)

    tic_fpr = fpr

En la gfica que muestra la curva ROC podemos observar que Laplace mejora el rendimiento en el caso de los datos de Tic Tac Toe, y sin embargo los empeora para German. Esto se debe a que la correccin de Laplace solo es positiva cuando queremos que un 0 no nos anule toda las probabilidades asociadas, cosa que ocurre con los datos de TicTacToe. Lo que ocurre en German es que no tenemos ceros, y por tanto manipula los resultados en vano, empeorando la precisin del algoritmo.
